# 精斗云云会计系统 - 启动说明

## 一、Docker 部署（推荐）

> 适合服务器、VPS 或任何安装了 Docker 的环境，**无需配置 Node.js 环境**。

### 前提条件
- Docker 20.10+
- Docker Compose v2（或 docker-compose v1.29+）

### 一键部署

```bash
# 1. 克隆项目
git clone https://github.com/Yangdongle668/Finance.git
cd Finance

# 2. 一键启动（首次约需 3-5 分钟构建镜像）
bash deploy.sh
```

完成后访问：**http://localhost**（或 `http://服务器IP`）

默认账号：`admin` / `Admin@123`

---

### deploy.sh 命令速查

```bash
bash deploy.sh              # 构建并启动（默认）
bash deploy.sh stop         # 停止服务（数据保留）
bash deploy.sh restart      # 重启服务
bash deploy.sh logs         # 实时查看所有日志
bash deploy.sh logs server  # 只看后端日志
bash deploy.sh logs client  # 只看 Nginx 日志
bash deploy.sh status       # 查看运行状态和访问地址
bash deploy.sh backup       # 备份数据库 → ./backups/
bash deploy.sh restore <文件> # 从备份恢复数据库
bash deploy.sh update       # 拉取新代码重新部署
bash deploy.sh clean        # ⚠️ 删除所有容器和数据
```

### 修改配置（端口/密钥）

编辑 `.env` 文件（首次运行自动从 `.env.example` 生成）：

```bash
APP_PORT=8080         # 改为你想要的端口（默认 80）
JWT_SECRET=xxx        # 生产环境务必修改为强密钥！
```

修改后重启：`bash deploy.sh restart`

## 二、本地开发启动

### 前提条件
- Node.js 18+
- npm 9+

### 第一次启动

```bash
# 安装依赖
cd server && npm install
cd ../client && npm install

# 初始化配置文件
cd ../server && cp .env.example .env

# 初始化数据库（创建管理员 + 标准科目表 + 当年期间）
npm run seed

# 启动后端（端口 3001）
npm run dev
```

新开一个终端：

```bash
cd client && npm run dev   # 前端 → http://localhost:5173
```

## 四、API 接口文档

### 认证

| Method | Path | 说明 |
|--------|------|------|
| POST | /api/auth/login | 登录，返回 JWT Token |
| GET | /api/auth/me | 获取当前用户信息 |
| POST | /api/auth/change-password | 修改密码 |
| GET | /api/auth/users | 用户列表（管理员） |
| POST | /api/auth/users | 新建用户（管理员） |

### 会计期间

| Method | Path | 说明 |
|--------|------|------|
| GET | /api/periods | 期间列表 |
| POST | /api/periods/ensure/:year | 初始化全年 12 个期间 |
| POST | /api/periods/:id/close | 结账 |
| POST | /api/periods/:id/reopen | 反结账（仅管理员） |

### 科目管理

| Method | Path | 说明 |
|--------|------|------|
| GET | /api/accounts | 科目列表 |
| GET | /api/accounts/leaf | 末级科目（凭证录入用） |
| POST | /api/accounts | 新增科目 |
| PATCH | /api/accounts/:code | 更新科目 |
| GET | /api/accounts/balances/:periodId | 期间科目余额 |
| GET | /api/accounts/dimensions/list | 核算项目（部门/客户等） |

### 凭证管理

| Method | Path | 说明 |
|--------|------|------|
| GET | /api/vouchers | 凭证列表（分页/多条件筛选） |
| POST | /api/vouchers | 新增凭证（含借贷平衡校验） |
| GET | /api/vouchers/:id | 凭证详情（含明细行） |
| POST | /api/vouchers/:id/submit | 提交审核 |
| POST | /api/vouchers/:id/approve | 审核通过 |
| POST | /api/vouchers/:id/reject | 驳回（退回草稿） |
| POST | /api/vouchers/:id/post | 记账（触发余额重算） |

| POST | /api/vouchers/:id/reverse | 红字冲销 |

|--------|------|------|

| GET | /api/reports/ledger/:accountCode | 明细账 |

## 五、扩展开发指南

### 新增模块标准步骤

```
1. server/src/domain/<模块>/types.ts        定义领域类型
2. server/src/infrastructure/repositories/  实现数据访问层
3. server/src/application/services/         实现业务逻辑层
4. server/src/api/routes/<模块>.ts          定义 REST 路由
5. server/src/server.ts                     注册路由
6. client/src/pages/<模块>/                创建前端页面
7. client/src/App.tsx                       注册路由
```

### 金额处理规范

| 位置 | 规范 |
|------|------|
| 数据库存储 | 以**分**（整数）存储，彻底避免浮点精度问题 |
| API 请求 | 前端传入**元**（小数），后端自动 × 100 转为分 |
| API 响应 | 以**元**（÷ 100）返回给前端 |

### 角色权限说明

| 角色 | 描述 |
|------|------|
| admin | 管理员，可反结账、管理用户 |
| supervisor | 主管，可审核凭证 |
| accountant | 会计，可录入/记账凭证 |
| cashier | 出纳，可录入收支凭证 |
| viewer | 只读，只能查看 |

## 六、技术栈

| 层次 | 技术选型 |
|------|----------|
| 前端框架 | React 18 + TypeScript |
| UI 组件库 | Ant Design 5 |
| 数据可视化 | ECharts 5 + echarts-for-react |
| 路由 | React Router v6 |
| 状态管理 | Zustand |
| HTTP 客户端 | Axios |
| 后端框架 | Express 4 + TypeScript |
| 数据库 | SQLite（better-sqlite3），可替换为 PostgreSQL |
| 认证 | JWT (jsonwebtoken) |
| 日志 | Winston |
| 构建工具 | Vite（前端）/ tsc（后端） |
| 容器化 | Docker + Docker Compose + Nginx |
